# Phase 1: Discovery - 初版需求提炼

## 需求来源

参考 [营营无忧.html](../../../营营无忧.html) 的标题、导航、场景与文案，提炼初版需求。

## 核心功能（从页面归纳）

### 1. 工作台布局

- **左侧固定侧栏**（240px）：
  - Logo区域：营营无忧品牌标识 + 副标题
  - 导航区域：首页、自动化（托管日历、门店经营、门店巡检、物流调度）、工具（市场、合同）
  - 对话历史：会话列表，支持新建/切换/删除
  - 用户区：用户头像、姓名、角色信息

- **主区**：
  - 标题区（main-header）：场景标题、副标题、创建对话按钮
  - 内容区（main-content）：根据场景切换不同视图

- **支持视图**：
  - 首页视图：欢迎语 + 操作指引（01-04）
  - 对话视图：消息列表（可滚动）+ 底部输入区

### 2. 导航与场景

- **首页**：操作指引（01-04）
  - 01: 左侧导航
  - 02: 托管日历
  - 03: 创建对话
  - 04: 模块内操作

- **自动化**：
  - 托管日历：按日开启/关闭各模块 AI 托管
  - 门店自动化：
    - AI门店经营：销售快览、待办、任务时间线、经营动态
    - AI门店巡检：巡检任务与报告
  - 运营自动化：
    - AI物流调度：运力与人力需求

- **工具栏**：
  - 市调选品分析
  - 合同智检

- **对话历史**：
  - 会话列表（可新建/切换/删除）
  - 历史标题展示（如"本月销售凭证与金蝶差异"）

### 3. 对话能力

- **创建会话**：点击"创建对话"按钮，新建会话
- **发送消息**：用户输入消息，支持快捷提问
- **流式展示**：助手回复以流式方式逐字/逐块展示
- **会话管理**：
  - 会话列表展示
  - 切换会话
  - 删除会话
  - 会话标题自动生成或手动编辑

- **消息样式**：
  - 用户气泡：右侧对齐，深色背景
  - AI回复：左侧对齐，卡片样式，支持操作按钮

### 4. 业务语义（用于 Mock 与后续 MCP）

- **门店经营**：销售额、客流、客单价、损耗率、待办任务
- **财务**：销售凭证、金蝶凭证差异
- **客流分析**：时段分析、趋势预测
- **物流**：运力需求、入库出库、人力需求
- **经营指标**：环比分析、同比分析
- **品类管理**：品类占比、汰换建议
- **定价**：定价参考、调价建议
- **补货**：补货建议、库存预警
- **市场**：市调选品分析
- **合同**：合同智检

**首版策略**：不接真实业务系统，由后端 Mock 工具返回示例数据。

## 用户与成功标准

### 用户

- 内部或演示用户
- 通过浏览器使用工作台与智能体对话

### 成功标准

1. **UI一致性**：前端展示与营营无忧风格一致
   - 深色主题（`--bg-main: #0f1419`）
   - 主色（`--accent: #00c896`）
   - 字体（DM Sans 主字体，JetBrains Mono 用于日期/数字）
   - 侧栏+主区布局

2. **对话功能**：
   - 能发起会话并发送消息
   - 后端流式返回（SSE），前端逐字/逐块展示
   - 会话列表与当前会话可切换

3. **Mock工具**：
   - 后端提供 1～2 个 Mock 工具（如「门店经营摘要」「金蝶凭证差异」）
   - Agent 可调用并返回 MOCK 数据

## 约束与待澄清

### 约束

- **后端**：Spring AI 2.0（Java 21+、Spring Boot 4.x）
- **前端**：主流框架（React + Vite）
- **业务系统/MCP**：先 Mock，不写死敏感配置

### 待澄清（Phase 3）

1. **首版范围**：是否只做「首页 + 对话」最小闭环，其余场景（托管日历、仪表盘等）是否作为静态占位页？
2. **Mock工具**：Mock 工具的数量与粒度？
3. **可观测性**：是否在首版接入可观测性（如 Langfuse）？
